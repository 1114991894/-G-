## 🚀 快速启动步骤

### 第一步：配置数据库
1. 确保MySQL数据库已安装并运行
2. 创建数据库：`bwg_performance`
3. 配置环境变量：
   ```bash
   cd api
   copy .env.example .env
   ```
4. 编辑 `.env` 文件，配置数据库连接信息：
   ```
   DB_HOST=localhost
   DB_PORT=3306
   DB_USER=root
   DB_PASSWORD=your_password
   DB_NAME=bwg_performance
   ```

### 第二步：安装依赖
```bash
# 安装后端依赖
cd api
npm install

# 安装客户端前端依赖
cd ../client/web
npm install

# 安装服务端前端依赖
cd ../../server/web
npm install
```

### 第三步：创建默认账户（重要！）
```bash
# 进入后端目录
cd api

# 运行种子数据脚本（会创建测试账户和示例数据）
npm run seed
```

**⚠️ 注意**:
- 此命令会**删除并重建**所有数据库表
- 仅用于开发和测试环境
- 生产环境请使用迁移脚本

### 第四步：启动服务

#### 方式一：分别启动（推荐用于开发）
```bash
# 终端1：启动后端API
cd api
npm run dev
# API服务运行在: http://localhost:3000

# 终端2：启动客户端前端
cd client/web
npm run dev
# 客户端运行在: http://localhost:5173

# 终端3：启动服务端前端
cd server/web
npm run dev
# 服务端运行在: http://localhost:5174
```

#### 方式二：使用Concurrent（推荐）
安装concurrently：
```bash
npm install -g concurrently
```

创建启动脚本（在项目根目录）：
```bash
concurrently \
  "cd api && npm run dev" \
  "cd client/web && npm run dev" \
  "cd server/web && npm run dev"
```

## ❓ 常见问题

### Q1: 运行种子数据时报错怎么办？
**A**: 检查以下几点：
1. MySQL数据库是否正常运行
2. `.env` 文件中的数据库配置是否正确
3. 数据库 `bwg_performance` 是否已创建
4. 端口3306是否被占用

### Q2: 登录时提示"用户名或密码错误"？
**A**: 
1. 确认是否已运行 `npm run seed` 创建测试账户
2. 检查用户名和密码是否正确（默认密码：123456）
3. 查看后端控制台是否有错误日志

### Q3: 前端无法访问后端API？
**A**: 
1. 确认后端API是否已启动（http://localhost:3000）
2. 检查CORS配置是否正确
3. 查看浏览器控制台的错误信息

### Q4: 忘记密码怎么办？
**A**: 
目前系统中"忘记密码"功能尚未实现。可以通过以下步骤重置：
1. 直接访问数据库
2. 找到 `users` 表
3. 手动更新对应用户的密码（需要使用bcrypt加密）

示例SQL（密码重置为123456）：
```sql
-- 需要先生成bcrypt哈希值
-- 可以使用在线工具：https://bcrypt-generator.com/
UPDATE users 
SET password = '$2b$10$YourHashedPasswordHere' 
WHERE username = 'admin';
```

### Q5: 如何创建新用户？
**A**: 有两种方式：
1. **通过注册页面**：访问登录页，点击"立即注册"
2. **通过后端API**：调用 `POST /api/v1/auth/register` 接口
